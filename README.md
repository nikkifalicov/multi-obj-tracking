Multi-object tracking library that's easy to use with Groundlight's Python SDK. Project was created under the supervision of an Machine Learning Engineer at Groundlight during a summer internship.

Groundlight created a platform to easily create custom-purpose Computer Vision models. The service allowed users to provide a query and upload images to easily obtain predictions. It included automatic model tuning and 24/7 data annotation by humans. While I worked at Groundlight, we encountered a problem: we needed to be able to accurately track multiple objects in real-time using the Groundlight platform. Groundlight already had support for running their models on-edge, but no pre-existing framework for instituting object tracking on top of the models.

Thus, my main task was to construct a library that could perform multi-object tracking. `sort.py`, `tracker_base.py`, `tracker_interface.py`, `type_definitions.py`, `track.py`, and the mock SDK provider were not written by me, but I did collaborate on creating them. The other tracking algorithms, their unit tests, and evaluation functionality were created by me. 

Groundlight no longer has a free tier available to the public, so code that can still be run will be enumerated below.

## Runnable Code
Environment setup: Package management was done through [uv](https://docs.astral.sh/uv/). To get started,make sure you have the `uv` package manager installed. See installation instructions [here](https://docs.astral.sh/uv/getting-started/installation/).

Then, you can install the package dependencies by running:

```shell
make install

# Or if you want to install the package with extra dev dependencies:
make install-dev
```

You can run tests by running 
```shell
# if you have access to a GPU
make test

# if you only want to run CPU tests
make test-cpu

# if you only want to run GPU tests
make test-gpu
```
You can run our formatting tools by running:

```shell
make format
```

Once your environment is set up, make sure your current working directory is set to the multi-obj-tracking repository. Aftewards, run

```shell
uv run sample_code.py
```
Which will print the Multi-Object Tracking Accuracy (MOTA) for the tracking algorithm.

A video called bytetrack_tracker_on_gl_inferences_demo.mp4 will be saved to your current working directory. It will allow you to see the tracks generated by the ByteTrack tracking algorithm on input from the MockSDK provider class (which simulates the output that would be produced by Groundlight)



## Algorithms implemented
Current tracking algorithms available are as follows:
1. SORTTracker, available via [sort.py](https://github.com/groundlight/tracking/blob/main/src/tracking/sort.py)
1. DeepSORTTracker, available via [deepsort.py](https://github.com/groundlight/tracking/blob/main/src/tracking/deepsort.py)
3. ByteTrackTracker, available via [bytetrack.py](https://github.com/groundlight/tracking/blob/main/src/tracking/bytetrack.py)
